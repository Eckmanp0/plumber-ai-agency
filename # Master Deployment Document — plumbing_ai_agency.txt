# Master Deployment Document — Plumber Voice Agent Agency

## Objective

Build a production-minded, multi-tenant plumber voice-agent platform using Vapi, Supabase, Google Calendar, Google Sheets, and SMS automation.

This system must support:

* automated onboarding
* multi-tenant dashboard
* calendar booking integration
* SMS reminders
* outbound AI follow-ups
* reusable deployment for multiple plumbing companies

---

## Core Stack

* **Vapi**: inbound and outbound voice agents, phone numbers, tool calling, dynamic variables
* **Supabase**: Postgres database, auth, row-level security, edge functions, storage
* **Google Calendar**: availability lookup and direct booking
* **Google Sheets**: per-client operational reporting and lead logs
* **SMS provider**: Twilio or equivalent for reminders and follow-ups
* **Frontend**: admin dashboard + client dashboard

---

## Product Modes

The platform must support two booking modes per client:

1. **request_only**

   * collect customer request
   * store requested windows
   * notify client for manual confirmation

2. **calendar_direct**

   * check live availability
   * offer appointment windows
   * book directly on connected calendar
   * trigger reminders automatically

## Multi-Tenant Database Model

Every tenant-scoped table must include `tenant_id`.

### 001_init.sql

```sql
create extension if not exists "pgcrypto";
create extension if not exists "uuid-ossp";

create table tenants (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  slug text unique not null,
  created_at timestamptz default now()
);

create table tenant_users (
  id uuid primary key default gen_random_uuid(),
  tenant_id uuid not null references tenants(id) on delete cascade,
  user_id uuid not null,
  role text not null check (role in ('agency_admin','client_admin','dispatcher','tech_viewer')),
  created_at timestamptz default now()
);

create table client_settings (
  id uuid primary key default gen_random_uuid(),
  tenant_id uuid not null references tenants(id) on delete cascade,
  business_name text not null,
  owner_name text,
  business_phone text,
  business_email text,
  service_area_json jsonb not null default '[]'::jsonb,
  services_json jsonb not null default '[]'::jsonb,
  hours_json jsonb not null default '{}'::jsonb,
  emergency_enabled boolean default false,
  same_day_allowed boolean default false,
  urgent_alert_phone text,
  reports_email text,
  booking_mode text not null default 'request_only' check (booking_mode in ('request_only','calendar_direct')),
  google_sheet_id text,
  vapi_assistant_id text,
  vapi_phone_number_id text,
  created_at timestamptz default now()
);

create table leads (
  id uuid primary key default gen_random_uuid(),
  tenant_id uuid not null references tenants(id) on delete cascade,
  caller_name text,
  caller_phone text,
  service_address text,
  issue text,
  service_type text,
  urgency text,
  preferred_time text,
  status text not null default 'new' check (status in ('new','booked','callback','declined','spam','needs_review')),
  source text default 'inbound_call',
  created_at timestamptz default now()
);

create table calls (
  id uuid primary key default gen_random_uuid(),
  tenant_id uuid not null references tenants(id) on delete cascade,
  lead_id uuid references leads(id) on delete set null,
  vapi_call_id text,
  direction text not null default 'inbound' check (direction in ('inbound','outbound')),
  transcript text,
  summary text,
  duration_seconds integer,
  outcome text,
  recording_url text,
  created_at timestamptz default now()
);

create table appointments (
  id uuid primary key default gen_random_uuid(),
  tenant_id uuid not null references tenants(id) on delete cascade,
  lead_id uuid references leads(id) on delete set null,
  external_event_id text,
  booked_start timestamptz,
  booked_end timestamptz,
  timezone text default 'America/Los_Angeles',
  status text not null default 'pending' check (status in ('pending','booked','confirmed','cancelled','needs_review')),
  notes text,
  created_at timestamptz default now()
);

create table monthly_usage (
  id uuid primary key default gen_random_uuid(),
  tenant_id uuid not null references tenants(id) on delete cascade,
  month text not null,
  total_calls integer default 0,
  qualified_leads integer default 0,
  booked_jobs integer default 0,
  minutes_used integer default 0,
  estimated_cost numeric default 0,
  invoice_amount numeric default 0,
  created_at timestamptz default now()
);
```

### 004_sms_and_followups.sql

```sql
create table reminders (
  id uuid primary key default gen_random_uuid(),
  tenant_id uuid not null references tenants(id) on delete cascade,
  appointment_id uuid not null references appointments(id) on delete cascade,
  reminder_type text not null check (reminder_type in ('sms','call')),
  scheduled_for timestamptz not null,
  status text not null default 'queued' check (status in ('queued','sent','failed','cancelled')),
  payload_json jsonb not null default '{}'::jsonb,
  created_at timestamptz default now()
);

create table sms_messages (
  id uuid primary key default gen_random_uuid(),
  tenant_id uuid not null references tenants(id) on delete cascade,
  lead_id uuid references leads(id) on delete set null,
  appointment_id uuid references appointments(id) on delete set null,
  to_phone text not null,
  direction text not null check (direction in ('outbound','inbound')),
  template_key text,
  body text not null,
  provider_message_id text,
  status text not null default 'queued',
  created_at timestamptz default now()
);

create table followup_rules (
  id uuid primary key default gen_random_uuid(),
  tenant_id uuid not null references tenants(id) on delete cascade,
  active boolean default true,
  trigger_reason text not null,
  followup_type text not null check (followup_type in ('sms','outbound_call')),
  delay_minutes integer not null,
  template_key text not null,
  created_at timestamptz default now()
);

create table followups (
  id uuid primary key default gen_random_uuid(),
  tenant_id uuid not null references tenants(id) on delete cascade,
  lead_id uuid references leads(id) on delete cascade,
  appointment_id uuid references appointments(id) on delete set null,
  followup_type text not null check (followup_type in ('sms','outbound_call')),
  trigger_reason text not null,
  scheduled_for timestamptz not null,
  status text not null default 'queued' check (status in ('queued','processing','sent','completed','failed','cancelled')),
  payload_json jsonb not null default '{}'::jsonb,
  created_at timestamptz default now()
);
```

### 005_rls_policies.sql

```sql
alter table tenant_users enable row level security;
alter table client_settings enable row level security;
alter table leads enable row level security;
alter table calls enable row level security;
alter table appointments enable row level security;
alter table reminders enable row level security;
alter table sms_messages enable row level security;
alter table followups enable row level security;
alter table monthly_usage enable row level security;
alter table calendar_connections enable row level security;

create policy "tenant can view own client settings"
on client_settings for select
using (
  tenant_id in (
    select tenant_id from tenant_users where tenant_users.user_id = auth.uid()
  )
);

create policy "tenant can view own leads"
on leads for select
using (
  tenant_id in (
    select tenant_id from tenant_users where tenant_users.user_id = auth.uid()
  )
);

create policy "tenant can insert own leads"
on leads for insert
with check (
  tenant_id in (
    select tenant_id from tenant_users where tenant_users.user_id = auth.uid()
  )
);

create policy "tenant can view own calls"
on calls for select
using (
  tenant_id in (
    select tenant_id from tenant_users where tenant_users.user_id = auth.uid()
  )
);

create policy "tenant can view own appointments"
on appointments for select
using (
  tenant_id in (
    select tenant_id from tenant_users where tenant_users.user_id = auth.uid()
  )
);
```

### Availability Flow

```text
Caller asks to book service
→ Vapi assistant collects preferred date/time
→ assistant calls calendar-availability
→ function returns 2–3 open slots
→ caller selects slot
→ assistant calls book_calendar_appointment
→ event created on Google Calendar
→ appointment row created in database
→ reminder rows created
→ client notified
```

## SMS Reminder Engine

### Reminder Schedule Defaults

* immediate confirmation after booking
* 24 hours before appointment
* 2 hours before appointment

### SMS Templates

#### booking confirmation

```text
Hi {{customer_name}}, you're scheduled with {{business_name}} on {{appointment_time}} at {{service_address}}. Reply C to confirm or call {{business_phone}}.
```

#### 24-hour reminder

```text
Reminder from {{business_name}}: your plumbing appointment is tomorrow at {{appointment_time}}. Reply if you need to reschedule.
```

#### on-the-way update

```text
{{business_name}} update: your technician is on the way. Questions? Call {{business_phone}}.
```

### Reminder Worker Flow

```text
appointment booked
→ insert queued reminders
→ worker polls reminders due now
→ sms-send function dispatches SMS
→ update sms_messages and reminders tables
→ mark sent or failed
```

## Outbound AI Follow-Ups

### Use Cases

* unbooked new lead after 15 minutes
* after-hours lead next morning
* estimate request not booked within 2 hours
* post-service review request after 24 hours

### Default Rule Seeds

* `unbooked_new_lead` → `sms` → 15 minutes
* `after_hours_callback` → `outbound_call` → 600 minutes
* `estimate_followup` → `sms` → 120 minutes
* `post_service_review` → `sms` → 1440 minutes

### Follow-Up Worker Flow

```text
new lead inserted
→ evaluate followup rules
→ queue followup rows
→ scheduler picks due followups
→ if sms, call sms-send
→ if outbound_call, call Vapi outbound endpoint
→ write results to followups and calls tables
```
